# Modern UX Guidance Skill

## CRAP Design Principles

### Contrast
- Create visual hierarchy through size, color, weight differences
- Headlines should be notably larger/bolder than body text
- Active states must be visually distinct from inactive
- CTAs should stand out from surrounding elements

### Repetition
- Reuse visual patterns throughout the interface
- Consistent button styles, card layouts, spacing
- Same interaction patterns for similar actions
- Unified icon style and sizing

### Alignment
- Every element should be visually connected to something else
- Use a grid system (8px or 4px base)
- Left-align text (avoid center for long content)
- Align form labels and inputs consistently

### Proximity
- Group related items together
- Use whitespace to separate unrelated groups
- Labels should be closer to their fields than to adjacent fields
- Action buttons near the content they affect

## Task-First UX
1. Identify the user's primary task on each screen
2. Make that task the most prominent element
3. Remove or de-emphasize everything that doesn't support the task
4. Reduce steps to completion
5. Provide clear next actions

## HCI Laws

### Jakob's Law
Users spend most time on OTHER sites — design consistently with conventions.

### Miller's Law
Chunk information into groups of 5-9 items.

### Doherty Threshold
System responses under 400ms feel instant. Over 1000ms, show loading indicators.

### Peak-End Rule
Users judge experiences by the peak moment and the end. Make completions feel rewarding.

### Von Restorff Effect
Items that stand out are more memorable. Use this for CTAs and key information.

### Zeigarnik Effect
People remember incomplete tasks. Use progress indicators to motivate completion.

## Interaction Psychology
- **Loss aversion**: Frame actions in terms of what users might lose, not just gain
- **Social proof**: Show what others have done (ratings, usage counts)
- **Anchoring**: Present the recommended option first
- **Default effect**: Set smart defaults — most users won't change them
- **Endowment effect**: Let users customize before committing

## Evaluation Checklist
When reviewing a design, ask:
- [ ] Can users complete their primary task in under 3 clicks?
- [ ] Is the visual hierarchy clear at a glance?
- [ ] Are error states helpful and actionable?
- [ ] Does it work without color (accessibility)?
- [ ] Is the loading experience graceful?
- [ ] Are empty states informative, not just blank?
- [ ] Is the mobile experience equivalent to desktop?
